## Description

The GET verification request is sent by Meta to your webhook endpoint whenever you set or edit the Callback URL or Verify token in the App Dashboard. Your endpoint must correctly respond to this request to be verified.

## Request Format

```
GET <CALLBACK_URL>?hub.mode=subscribe&hub.challenge=<RANDOM>&hub.verify_token=<YOUR_TOKEN>
```

## Query Parameters

| Parameter | Description | Example |
|-----------|-------------|---------|
| `hub.mode` | Always `subscribe` | `subscribe` |
| `hub.challenge` | Random string generated by Meta | `1158201444` |
| `hub.verify_token` | Your configured verify token | `vibecoding` |

## Required Response

### On Valid Request (token matches)
- **Status**: `200`
- **Body**: The exact `hub.challenge` value

### On Invalid Request (token doesn't match)
- **Status**: `403` (or any non-200)
- **Body**: Anything (ignored)

## Implementation Example

```javascript
app.get('/', (req, res) => {
  const mode = req.query['hub.mode'];
  const challenge = req.query['hub.challenge'];
  const token = req.query['hub.verify_token'];
  
  if (mode === 'subscribe' && token === process.env.VERIFY_TOKEN) {
    res.status(200).send(challenge);
  } else {
    res.status(403).end();
  }
});
```
